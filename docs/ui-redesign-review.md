# UIリデザイン レビュー指摘事項

レビュー日: 2026-04-11

## 変更概要

- 2カラム（設定左/プレビュー右）→ 1カラム縦フローに変更
- LLM連携「おまかせで選ぶ」ボタンを新設（placeholder）
- フォント状態を PreviewGrid 内部 → App.tsx に lift up
- 6フォント同時Canvas描画プレビューは維持

## 指摘事項

### 優先度: 高

#### 1. `CANVAS_SIZE` の重複定義
- `App.tsx` と `PreviewGrid.tsx` の両方で `128` をハードコードしている
- 片方を変えたらもう片方を変え忘れるリスクがある
- **対応**: 定数ファイルに集約して import する

#### 2. 旧コンポーネントがデッドコードとして残存
- `Preview.tsx`, `PreviewThumbnail.tsx` がどこからも import されていない
- `types/index.ts` の `EmojiSettings`, `CanvasSize` も旧コンポーネント専用
- **対応**: 削除して整理。types は新しい設計に合わせて更新

#### 3. ダウンロード時のファイル名サニタイズ
- テキストに改行が含まれる場合（例: `"こん\nにちは"`）、そのままファイル名になると OS によっては問題が起きる
- **対応**: `text.replace(/\n/g, '_')` 等でサニタイズ

### 優先度: 中

#### 4. ダウンロードロジックの重複
- `App.tsx` に直書きされている Canvas → blob → ダウンロード処理は旧 `Preview.tsx` と重複
- **対応**: `utils/downloadEmoji.ts` 等に切り出し

#### 5. 定数の配置
- `COLOR_OPTIONS`, `FONTS` が `App.tsx` から export されている
- 設定データはコンポーネントの責務ではない
- **対応**: `constants.ts` 等に分離

#### 6. 型定義の分散
- `ColorOption` が `SettingsPanel.tsx`、`Font` が `PreviewGrid.tsx` で個別定義されている
- **対応**: `types/index.ts` に集約

### 優先度: 低

#### 7. コメント番号の欠番
- App.tsx のセクションコメントが `①` → `③` → `④` で `②` が抜けている

#### 8. 高解像度ディスプレイ対応
- 旧 `Preview.tsx` は `devicePixelRatio` を使った高解像度対応をしていたが、新 `FontCard` では省略されている
- Canvas を 128px で描画し CSS で 80px に縮小しているため結果的に少し高精細だが、意図的な対応ではない
- **対応**: 方針を決めて明示する

## 全体評価

リデザインの方向性は良い。操作フローが直線的になって分かりやすくなっている。
主な課題は「旧コードの残骸の片付け」と「定数・型定義の整理」で、設計上の大きな問題はない。
